# VEngine Scene Serialization

## Purpose

The first scene persistence path is handwritten JSON. It intentionally does not depend on reflection yet, so the schema
only covers built-in first-stage component types:

- `TransformComponent`
- `MeshRenderComponent`
- `CameraComponent`
- `LightComponent`

JSON generated by `SceneSerialization` is pretty-printed with four-space indentation through `JsonUtils`. Scalar
arrays, such as vectors and quaternions, stay on one line.

## File Shape

Scene files use the `.vescene` extension:

```json
{
    "schemaVersion": 1,
    "name": "Sample Scene",
    "rootGameObjects": []
}
```

Each GameObject stores a name, an ordered component list, and child GameObjects. Hierarchy ownership follows the runtime
model: children are serialized under the owning `TransformComponent`'s GameObject.

```json
{
    "name": "Cube",
    "components": [
        {
            "type": "TransformComponent",
            "enabled": true,
            "localPosition": [0, 0, 0],
            "localRotation": [0, 0, 0, 1],
            "localScale": [1, 1, 1]
        }
    ],
    "children": []
}
```

## Component Fields

`TransformComponent`:

- `enabled`
- `localPosition`
- `localRotation`
- `localScale`

`MeshRenderComponent`:

- `enabled`
- `mesh.assetID`
- `material.assetID`
- `boundsCenter`
- `boundsExtents`

Asset references are serialized as `AssetID` values:

```json
{
    "type": "MeshRenderComponent",
    "enabled": true,
    "mesh": {
        "assetID": {
            "guid": "11111111-1111-1111-1111-111111111111",
            "subID": 0
        }
    },
    "material": {
        "assetID": {
            "guid": "22222222-2222-2222-2222-222222222222",
            "subID": 0
        }
    },
    "boundsCenter": [0, 0, 0],
    "boundsExtents": [0.5, 0.5, 0.5]
}
```

Readers accept legacy `meshAssetGuid` and `materialAssetGuid` fields and convert them to `AssetID(guid, 0)`. Writers
emit only the structured `AssetID` shape.

`CameraComponent`:

- `enabled`
- `primary`
- `projectionMode`, currently `Perspective` or `Orthographic`
- `verticalFieldOfViewRadians`
- `orthographicSize`
- `aspectRatio`
- `nearClipPlane`
- `farClipPlane`

`LightComponent`:

- `enabled`
- `lightType`, currently `Directional` or `Point`
- `color`
- `intensity`
- `range`
- `innerConeAngleRadians`
- `outerConeAngleRadians`
- `castShadows`

## Runtime Notes

`SceneSerialization` is a data-only string serialization layer. It does not read files, request resources, or own the
complete scene construction flow.

`SceneSystem` owns live scene construction. It requests a `SceneResource`, deserializes the scene data, binds component
`AssetRef` values through `ResourceSystem`, and then rebuilds the render-thread scene. Bound `AssetRef` handles own their
resource references; scene unload releases those references by destroying or resetting the components that hold them.
