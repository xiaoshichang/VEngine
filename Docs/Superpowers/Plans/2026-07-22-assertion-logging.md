# Assertion Logging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist every post-initialization assertion to the VEngine log before its debug break and include D3D12 validation details in failed pipeline diagnostics.

**Architecture:** Keep `Core/Assert` independent and install a logging-owned assertion handler for the lifetime of initialized logging. The handler emits one fatal record, flushes it, and has a recursion-safe emergency fallback. D3D12 captures InfoQueue messages generated by failed root-signature and PSO calls into the backend error already reported by `ShaderManager`.

**Tech Stack:** C++20, VEngine logging facade, Boost.Log, D3D12 InfoQueue, CMake/CTest, MSVC.

- [ ] **Step 1: Add the focused assertion logging test**

Create a standalone test that initializes logging to a test-local file, reports an assertion without invoking a macro debug break, shuts logging down, and checks the persisted record:

```cpp
#include "Engine/Runtime/Core/Assert.h"
#include "Engine/Runtime/Logging/Log.h"

#include <filesystem>
#include <fstream>
#include <iostream>
#include <iterator>
#include <string>

namespace
{
    bool Expect(bool condition, const char* message)
    {
        if (!condition)
        {
            std::cerr << "FAILED: " << message << '\n';
        }
        return condition;
    }
}

int main()
{
    const std::filesystem::path logPath = std::filesystem::current_path() / "VEngineAssertionLoggingTests.log";
    std::error_code errorCode;
    std::filesystem::remove(logPath, errorCode);

    ve::LoggingConfig config;
    config.minimumSeverity = ve::LogSeverity::Trace;
    config.enableConsole = false;
    config.enableFile = true;
    config.enableDebuggerOutput = false;
    config.filePath = logPath;
    if (!Expect(ve::InitializeLogging(config) == ve::ErrorCode::None, "Logging should initialize"))
    {
        return 1;
    }

    const ve::AssertionInfo assertion{"syntheticExpression", "synthetic assertion message", "SyntheticAssertion.cpp", "SyntheticFunction", 42};
    ve::ReportAssertionFailure(assertion);
    ve::ShutdownLogging();

    std::ifstream stream(logPath, std::ios::binary);
    const std::string contents((std::istreambuf_iterator<char>(stream)), std::istreambuf_iterator<char>());
    const bool passed = Expect(contents.find("[Fatal][Assert]") != std::string::npos, "Assertion should use Fatal/Assert") &&
                        Expect(contents.find("Expression: syntheticExpression") != std::string::npos, "Expression should be logged") &&
                        Expect(contents.find("Message: synthetic assertion message") != std::string::npos, "Message should be logged") &&
                        Expect(contents.find("File: SyntheticAssertion.cpp:42") != std::string::npos, "Source location should be logged") &&
                        Expect(contents.find("Function: SyntheticFunction") != std::string::npos, "Function should be logged");
    stream.close();
    std::filesystem::remove(logPath, errorCode);
    return passed ? 0 : 1;
}
```

## Task 2: Install a recursion-safe logging assertion handler

**Files:**

- Modify: `Engine/Runtime/Logging/Log.h`
- Modify: `Engine/Runtime/Logging/Log.cpp`
- Test: `Tests/Unit/AssertionLoggingTests.cpp`

- [ ] **Step 1: Expose explicit flushing**

Add this public facade declaration to `Log.h`:

```cpp
/// Flushes every initialized logging sink before returning.
void FlushLogging() noexcept;
```

Implement it by checking initialized state under `gLoggingMutex`, then calling `boost::log::core::get()->flush()` inside a catch-all boundary.

- [ ] **Step 2: Add formatting and emergency fallback**

In `Log.cpp`, include `Assert.h` and add logging-private helpers that produce this payload:

```text
VEngine assertion failed
Expression: <expression>
Message: <message>
File: <file>:<line>
Function: <function>
```

The emergency helper must use `std::fprintf(stderr, ...)` with the raw `AssertionInfo` fields and also call the existing debugger-output helper on Windows. It must not call `LogMessage`, callbacks, or Boost.Log.

- [ ] **Step 3: Implement the logging-owned handler**

Use a thread-local recursion guard and a catch-all boundary:

```cpp
void LoggingAssertionHandler(const AssertionInfo& info) noexcept
{
    thread_local bool handlingAssertion = false;
    if (handlingAssertion)
    {
        WriteEmergencyAssertionOutput(info);
        return;
    }

    handlingAssertion = true;
    try
    {
        const std::string message = FormatAssertionMessage(info);
        LogMessage(LogSeverity::Fatal, "Assert", message);
        FlushLogging();
    }
    catch (...)
    {
        WriteEmergencyAssertionOutput(info);
    }
    handlingAssertion = false;
}
```

- [ ] **Step 4: Tie handler lifetime to logging lifetime**

After all sinks and `gLoggingState` are successfully initialized, call:

```cpp
SetAssertionHandler(LoggingAssertionHandler);
```

At the beginning of `ShutdownLogging()`, before removing sinks or marking logging uninitialized, call:

```cpp
SetAssertionHandler(nullptr);
```

If initialization throws or fails, leave the default stderr handler installed.

## Task 3: Preserve D3D12 validation messages in backend errors

**Files:**

- Modify: `Engine/RHI/D3D12/D3D12Rhi.cpp`

- [ ] **Step 1: Add operation-local InfoQueue collection**

Add private `D3D12Device` helpers with these contracts:

```cpp
[[nodiscard]] uint64_t GetStoredDebugMessageCount() const noexcept;
[[nodiscard]] std::string BuildD3D12FailureMessage(const char* operation, HRESULT result, uint64_t firstMessageIndex) const;
```

`GetStoredDebugMessageCount()` returns zero without `infoQueue_`. `BuildD3D12FailureMessage()` starts with `MakeHResultError`, iterates from `firstMessageIndex` to `GetNumStoredMessagesAllowedByRetrievalFilter()`, retrieves each `D3D12_MESSAGE` with the two-call size/query pattern, and appends message ID plus description. Retrieval failure skips only that message.

- [ ] **Step 2: Capture root-signature and PSO failures**

Immediately before `ID3D12Device::CreateRootSignature` and `ID3D12Device::CreateGraphicsPipelineState`, record the current message count. On failure, replace the generic HRESULT-only error with:

```cpp
SetLastError(BuildD3D12FailureMessage("ID3D12Device::CreateGraphicsPipelineState", result, firstMessageIndex));
```

Use the corresponding operation name for root-signature creation. Root-signature serialization continues using its existing compiler error blob.

- [ ] **Step 3: Report callback registration failure**

When `RegisterMessageCallback` fails, emit:

```cpp
VE_LOG_WARN_CATEGORY("D3D12", "Failed to register the D3D12 debug message callback: HRESULT=0x{:08X}.", static_cast<unsigned>(callbackResult));
```

Keep `infoQueue_` alive so stored-message capture still works without the callback.
